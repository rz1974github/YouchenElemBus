# 松山車站到玉成國小公車動態

## 目標
做一個手機可用的 2D 公車動態畫面，只接即時 API，不存歷史、不做帳號、不做報表。畫面聚焦在松山車站到玉成國小這段短路線，去程與回程共用同一套路線圖與視覺規則。

## 核心規則
- 前端採 React + PWA。
- 路線圖用 SVG 表現。
- 即時資料來源以 TDX 為主。
- 先用 polling 更新，每 5 秒更新一次，不先做資料庫。
- 程式進入畫面時預設是去程，使用者可在畫面上切換去／回程。

## 資料結構
### appsettings.json
只放靜態資料與門檻值。
- `timeThresholdMinutes`: 10
- `pollingIntervalSeconds`: 5
- `busNumbers`: 公車號碼陣列
- `stationNames`: 單一站名陣列

站名只保留一份：
- 去程讀取 `0 ... n-1`
- 回程讀取 `n-1 ... 0`

### 路線與站點
路線使用抽象等距圖，不追求真實地圖比例。
- 站名固定在畫面左側
- 站與站之間等距
- 以圓形與線段表示站點與路徑
- 去程與回程共用同一套圖形語言，只是方向相反

## 畫面規則
### 去程
- 預設進入畫面先顯示去程
- 車子視覺上從下往上移動
- 站名依正序顯示

### 回程
- 使用者切到回程後顯示回程
- 車子視覺上從上往下移動
- 站名依反序讀取同一份 `stationNames`

### 多車顯示
- 以 `10` 分鐘為門檻
- 同方向若有多台車都在 `10` 分鐘內，就在右側依序增加平行線
- 每多一台車就多一條線與一組圓框時間資訊
- 每條線頂端顯示車號
- 線與站點同水平的位置顯示到站時間
- 車子到站時，車子圖示貼在對應站點同水平位置
- 超過終點站後消失

## 驗證標準
- 10 分鐘內多車時會同時顯示多條平行線。
- 車輛超過終點站後會消失。
- 去回程切換時，路線與站名會反向正確顯示。
- 手機尺寸下仍可讀。
- PWA 可安裝，且基本載入正常。
